Why Logging Matters

Logging is the process of recording events, errors, and informational messages from your program. Good logging is essential for debugging, monitoring, and maintaining real-world applications‚Äîespecially as they grow in size and complexity.

Unlike print() statements, logging can be configured to record messages at different severity levels, write to files, send alerts, and integrate with monitoring tools. It helps you understand what your code is doing, track down bugs, and even audit user actions.

Real-world scenarios:

  • Debugging issues in production without stopping the app
  • Tracking user activity and errors in web servers
  • Auditing financial transactions or security events
  • Monitoring long-running data pipelines or background jobs
  • Integrating with tools like Sentry, ELK stack, or cloud logging services

Python's built-in logging module is powerful, flexible, and used in nearly every professional Python project.

üí°

Key Concepts: Logging in Python

  • Log Levels: Control the importance of messages. Common levels: DEBUG, INFO, WARNING, ERROR, CRITICAL.
  • Handlers: Decide where logs go (console, file, email, etc.).
  • Formatters: Control how log messages look (add timestamps, module names, etc.).
  • Logger Hierarchy: Supports modular logging for large projects (e.g., app.module.submodule).

Basic Logging Example

basic_logging.py 
import logging

logging.basicConfig(level=logging.INFO)
logger = logging.getLogger('myapp')

logger.info('This is an info message')
logger.warning('This is a warning')
logger.error('This is an error')

üí° Expected Output:

INFO:root:This is an info message.
WARNING:root:This is a warning.
ERROR:root:This is an error!

Customizing Log Format and Handlers

custom_logging.py 
import logging

logger = logging.getLogger("myapp")
handler = logging.StreamHandler()
formatter = logging.Formatter('%(asctime)s %(levelname)s [%(name)s]: %(message)s')
handler.setFormatter(formatter)
logger.addHandler(handler)
logger.setLevel(logging.DEBUG)

logger.debug("Debug message")
logger.info("Info message")
logger.error("Error message")

üí° Expected Output:

2024-05-01 12:00:00,000 DEBUG [myapp]: Debug message
2024-05-01 12:00:00,001 INFO [myapp]: Info message
2024-05-01 12:00:00,002 ERROR [myapp]: Error message

üèÜ Project Challenge: Multi-Module Logging System

Real-World Project: Design and implement a robust logging system for a multi-module Python application (e.g., a web server, data pipeline, or microservice). This project requires careful planning and creative problem-solving‚Äîthink about scalability, maintainability, and real-world needs.

  • Define a logger hierarchy (e.g., myapp.api, myapp.db, myapp.utils).
  • Configure different handlers and formatters for each module (e.g., errors to errors.log, info to app.log).
  • Support log rotation (e.g., using RotatingFileHandler or TimedRotatingFileHandler).
  • Integrate with a third-party service (e.g., email alerts for critical errors, or sending logs to a cloud service).
  • Document your design: explain your choices, how to add new modules, and how to test the logging system.
  • Bonus: Add contextual information (user IDs, request IDs, etc.) to your logs for better traceability.
