41. Type Hints and Annotations

Advanced Level
üî§

Making Your Code More Predictable

Python is a dynamically typed language, meaning you don't declare the type of a variable when you create it. While this offers flexibility, it can sometimes make code harder to read and debug, especially in larger projects. Type Hints (or Type Annotations) were introduced in Python 3.5 to address this.

Type hints are not enforced by the Python interpreter at runtime, but they provide valuable metadata that can be used by:

  • Other developers: To understand what type of data a function expects or returns.
  • IDEs and Linters: To provide better auto-completion, refactoring, and static analysis (catching potential type-related bugs before running the code).
  • Type Checkers (e.g., MyPy): External tools that can analyze your code and report type inconsistencies.
üí°

Syntax for Type Hints

Type hints are added using a colon (:) after the variable name, parameter, or before the return arrow (->) for functions.

1. Variables:

type_hint_variables.py 
name: str = "Alice"
age: int = 30
price: float = 19.99
is_active: bool = True

2. Function Parameters and Return Values:

type_hint_functions.py 
def greet(name: str) -> str:
    """Greets a person by name."""
    return f"Hello, {name}!"

def add_numbers(a: int, b: int) -> int:
    """Adds two integers and returns their sum."""
    return a + b

def calculate_average(numbers: list[float]) -> float:
    """Calculates the average of a list of floats."""
    return sum(numbers) / len(numbers)

# Example usage
print(greet("Bob"))
print(add_numbers(10, 20))
print(calculate_average([10.0, 20.0, 30.0]))

3. Using typing Module for Complex Types:

For more complex types like lists, dictionaries, or optional values, you'll often import types from the built-in typing module.

type_hint_typing.py 
from typing import List, Dict, Optional, Union

def process_data(data: Dict[str, Union[str, int]]) -> List[str]:
    """
    Processes a dictionary and returns a list of formatted strings.
    data: a dictionary where keys are strings, and values can be strings or integers.
    """
    results: List[str] = []
    for key, value in data.items():
        results.append(f"{key}: {value}")
    return results

def get_user_id(username: str) -> Optional[int]:
    """
    Returns a user ID (int) if found, otherwise None.
    Optional[int] means the function might return an int or None.
    """
    user_ids = {"alice": 101, "bob": 102}
    return user_ids.get(username)

# Example usage
user_profile = {"name": "Charlie", "age": 25}
formatted_info = process_data(user_profile)
print("Formatted Info:", formatted_info)

print("Alice's ID:", get_user_id("alice"))
print("Eve's ID:", get_user_id("eve"))

üí° Expected Output:

Formatted Info: ['name: Charlie', 'age: 25']
Alice's ID: 101
Eve's ID: None
