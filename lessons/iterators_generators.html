Understanding Iterators

In Python, an iterator is an object that represents a stream of data. It allows you to traverse through a sequence of elements one by one, without needing to know the underlying structure of the sequence. Many built-in types like lists, tuples, strings, and dictionaries are iterable, meaning you can get an iterator from them.

Iterators implement two special dunder methods:

  • __iter__(self): Returns the iterator object itself.
  • __next__(self): Returns the next item from the iteration. If there are no more items, it must raise a StopIteration exception.

When you use a for loop in Python, it implicitly uses the iterator protocol.

Introducing Generators

Generators are a simpler and more memory-efficient way to create iterators. They are functions that, instead of returning a single value and exiting, use the yield keyword to produce a sequence of values one at a time.

When a generator function is called, it returns a generator object (which is an iterator). The code in the function doesn't run immediately. It only runs when next() is called on the generator object (e.g., implicitly by a for loop).

  • Memory Efficiency: Generators produce items on demand (lazily), which means they don't store the entire sequence in memory. This is extremely useful when dealing with very large or infinite sequences.
  • Simplicity: Writing a generator function is often much simpler than manually creating a class that implements the iterator protocol.

Iterators and Generators in Action

üí°

Example: Custom Iterator and Simple Generator

iterator_generator_demo.py 
# --- Custom Iterator Class ---
class MyRange:
    def __init__(self, start, end):
        self.current = start
        self.end = end

    def __iter__(self):
        return self

    def __next__(self):
        if self.current < self.end:
            num = self.current
            self.current += 1
            return num
        else:
            raise StopIteration

print("--- Using Custom Iterator (MyRange) ---")
for i in MyRange(1, 4):
    print(i)

# --- Simple Generator Function ---
def countdown(n):
    print("Starting countdown...")
    while n > 0:
        yield n # Yield pauses execution and returns a value
        n -= 1
    print("Countdown finished!")

print("\n--- Using Generator (countdown) ---")
gen = countdown(3)
print(f"Generator object: {gen}") # This is the iterator object

print(f"Next value: {next(gen)}") # First yield
print(f"Next value: {next(gen)}") # Second yield

print("\nContinuing with for loop:")
for val in gen: # Continues from where it left off
    print(val)
