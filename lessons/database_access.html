Key Concepts: Database Access in Python

  • Database Types: Relational (SQL: SQLite, MySQL, PostgreSQL) vs. NoSQL (MongoDB, Redis, etc.)
  • Connections & Cursors: Open a connection, create a cursor, execute queries, and close connections.
  • Connection Pooling: Reuse connections for performance in web apps.
  • Parameterized Queries: Prevent SQL injection by using placeholders.
  • Transactions: Use commit() and rollback() for data integrity.
  • Fetching Data: Use fetchone(), fetchall(), and iterate over results.
  • Context Managers: Use with for safe connection/cursor handling.
  • Migrations: Evolve your schema safely (Alembic, Django ORM, etc.).
  • ORMs: Libraries like SQLAlchemy provide higher-level abstractions.
  • Security: Always validate input and use least-privilege DB users.

Database Normalization & Schema Design

Good database design prevents data duplication, ensures consistency, and makes queries efficient. Normalization is the process of structuring tables to reduce redundancy and improve integrity. Most production databases use at least 3rd Normal Form (3NF).

  • 1NF: Each field contains only atomic values (no lists or sets).
  • 2NF: Every non-key column depends on the whole primary key.
  • 3NF: No transitive dependencies (non-key columns depend only on the key).

Schema design also includes choosing data types, indexes, and relationships (one-to-many, many-to-many).

Indexes & Performance

Indexes speed up queries by allowing the database to find rows faster, but they add overhead to writes. Use indexes on columns you search or join on frequently, but avoid over-indexing.

  • Primary keys are indexed by default.
  • Composite indexes can speed up multi-column searches.
  • Use EXPLAIN to analyze query plans and optimize slow queries.

Advanced SQL: Joins, Subqueries, Window Functions

SQL is a powerful language for data analysis. Mastering joins, subqueries, and window functions lets you write complex, efficient queries.

  • Joins: Combine rows from multiple tables (INNER, LEFT, RIGHT, FULL).
  • Subqueries: Use a query inside another query for filtering or aggregation.
  • Window Functions: Perform calculations across sets of rows (e.g., running totals, ranking).
advanced_sql.sql 
-- Join example
SELECT u.name, o.amount
FROM users u
JOIN orders o ON u.id = o.user_id;

-- Window function example
SELECT name, amount, SUM(amount) OVER (ORDER BY amount) AS running_total
FROM orders;

SQL vs NoSQL: When to Use Each

SQL databases (like PostgreSQL, MySQL, SQLite) are best for structured data, complex queries, and strong consistency. NoSQL databases (like MongoDB, Redis, Cassandra) are better for unstructured or rapidly changing data, horizontal scaling, and flexible schemas.

  • Use SQL for financial data, analytics, and apps needing ACID guarantees.
  • Use NoSQL for big data, caching, real-time analytics, or when schema changes often.
  • Many modern systems use both: SQL for core data, NoSQL for logs, caching, or search.

Database Scaling & High Availability

  • Vertical scaling: Add more resources (CPU, RAM) to a single server.
  • Horizontal scaling: Add more servers (sharding, replication, distributed databases).
  • Replication: Keep copies of data on multiple servers for redundancy and read performance.
  • Sharding: Split data across servers for write scalability.
  • CAP theorem: Consistency, Availability, Partition tolerance‚Äîchoose two.
üîí

Database Security & Backups

  • Encryption at rest and in transit (SSL/TLS).
  • Access control: Use least-privilege users and roles.
  • Regular backups and tested restore procedures.
  • Audit logging and monitoring for suspicious activity.

Example: SQLite - Creating and Querying a Table

sqlite_example.py 
import sqlite3

conn = sqlite3.connect(":memory:")
cur = conn.cursor()
cur.execute("CREATE TABLE users (id INTEGER, name TEXT)")
cur.execute("INSERT INTO users VALUES (?, ?)", (1, "Alice"))
cur.execute("INSERT INTO users VALUES (?, ?)", (2, "Bob"))
cur.execute("SELECT * FROM users")
rows = cur.fetchall()
print(rows)
conn.close()
Explanation: This code imports the sqlite3 module and creates an in-memory database. It then creates a cursor, defines a users table, and inserts two users (Alice and Bob) using parameterized queries. It selects all users from the table, fetches the results, prints them, and finally closes the connection.

Example: Parameterized Queries & SQL Injection

sql_injection_example.py 
import sqlite3

conn = sqlite3.connect(":memory:")
cur = conn.cursor()
cur.execute("CREATE TABLE users (id INTEGER, name TEXT)")
cur.execute("INSERT INTO users VALUES (?, ?)", (1, "Alice"))

# BAD: Vulnerable to SQL injection
user_input = "1 OR 1=1"
query = f"SELECT * FROM users WHERE id = {user_input}"
print(cur.execute(query).fetchall())

# GOOD: Use parameterized queries
cur.execute("SELECT * FROM users WHERE id = ?", (1,))
print(cur.fetchall())
Explanation: This code first creates a table and inserts a user. It then demonstrates a vulnerable query using string formatting, which could allow SQL injection if user input is malicious. The second query uses a parameterized statement, which safely passes the value and prevents injection. Both results are printed.

üí° Expected Output:

[(1, 'Alice')]

Example: Transactions and Error Handling

transactions_example.py 
import sqlite3

try:
    conn = sqlite3.connect(":memory:")
    cur = conn.cursor()
    cur.execute("CREATE TABLE test (id INTEGER)")
    cur.execute("INSERT INTO test VALUES (1)")
    conn.commit()
except Exception as e:
    conn.rollback()
    print(e)
finally:
    conn.close()
Explanation: This code tries to connect to an in-memory SQLite database, create a table, and insert a value. If any error occurs, it rolls back the transaction and prints the error. The finally block ensures the connection is closed whether or not an error happened.

Example: Context Managers for Connections/Cursors

context_manager_example.py 
import sqlite3

with sqlite3.connect(":memory:") as conn:
    cur = conn.cursor()
    cur.execute("CREATE TABLE t (id INTEGER)")
    cur.execute("INSERT INTO t VALUES (1)")
    print(cur.execute("SELECT * FROM t").fetchall())
Explanation: This code uses a with statement to open a SQLite connection, which ensures the connection is closed automatically. It creates a table, inserts a value, and prints all rows from the table. The context manager handles cleanup even if an error occurs.

Example: ORM with SQLAlchemy

sqlalchemy_example.py 
# Requires: pip install sqlalchemy
from sqlalchemy import create_engine, Column, Integer, String, Base

engine = create_engine('sqlite:///:memory:')

class User(Base):
    __tablename__ = 'users'
    id = Column(Integer, primary_key=True)
    name = Column(String)

Base.metadata.create_all(engine)
Explanation: This code imports SQLAlchemy components, creates an in-memory SQLite engine, and defines a User class mapped to a users table. The table has id and name columns. Base.metadata.create_all(engine) creates the table in the database.

‚úÖ Best Practices for Database Access

  • Always use parameterized queries to prevent SQL injection.
  • Close connections and cursors using with or finally blocks.
  • Use connection pooling for web apps (e.g., SQLAlchemy or mysql-connector pooling).
  • Handle exceptions and roll back transactions on errors.
  • Use migrations to manage schema changes safely.
  • Grant the minimum required privileges to database users.
  • Validate and sanitize all user input.

‚ö†Ô∏è Common Pitfalls

  • Forgetting to close connections (can exhaust DB resources).
  • Using string formatting for SQL queries (risk of SQL injection).
  • Not handling exceptions or rolling back on errors.
  • Assuming all databases use the same SQL dialect.
  • Data type mismatches between Python and SQL.
  • Hardcoding credentials in code (use environment variables or config files).
