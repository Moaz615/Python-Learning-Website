Why Multithreading Matters

Multithreading allows a program to run multiple operations concurrently within the same process. In Python, threads are useful for I/O-bound tasks (like network requests or file operations) and for keeping applications responsive.

While Python's Global Interpreter Lock (GIL) limits true parallelism for CPU-bound tasks, multithreading is still powerful for many real-world scenarios.

Real-world scenarios:

  • Downloading files or web pages in parallel
  • Handling multiple client connections in a server
  • Performing background tasks while keeping a UI responsive
  • Monitoring multiple sensors or data streams

Python's threading module makes it easy to create and manage threads.

💡

Key Concepts: Python Multithreading

  • Thread: A lightweight, independent unit of execution within a process.
  • GIL (Global Interpreter Lock): A mutex that allows only one thread to execute Python bytecode at a time. Limits CPU-bound parallelism, but not I/O-bound concurrency.
  • threading.Thread: The main class for creating and managing threads in Python.
  • start() and join(): start() begins thread execution; join() waits for a thread to finish.
  • Race Condition: A bug that occurs when threads access shared data concurrently without proper synchronization.

Example: Basic Multithreading

threading_example.py 
import threading
import time

def worker(name):
    print(f"Thread {name} starting")
    time.sleep(2)
    print(f"Thread {name} finished")

threads = []
for i in range(3):
    t = threading.Thread(target=worker, args=(i,))
    threads.append(t)
    t.start()

for t in threads:
    t.join()

print("All threads complete")

💡 Expected Output:

Thread 0 starting
Thread 1 starting
Thread 2 starting
Thread 0 finished
Thread 1 finished
Thread 2 finished
All threads complete

Example: Race Condition

race_condition.py 
import threading

counter = 0

def increment():
    global counter
    temp = counter
    temp += 1
    counter = temp

threads = []
for _ in range(1000):
    t = threading.Thread(target=increment)
    threads.append(t)
    t.start()

for t in threads:
    t.join()

print("Counter:", counter)  # Not always 1000!

💡 Expected Output:

Counter: (not always 1000)
