Why Multiprocessing Matters

Multiprocessing allows Python programs to run multiple processes in parallel, taking full advantage of multi-core CPUs. This is essential for CPU-bound tasks that are limited by the Global Interpreter Lock (GIL) in standard threading.

Real-world scenarios:

  • Speeding up data processing or scientific computations
  • Running simulations or batch jobs in parallel
  • Handling large-scale web scraping or automation tasks

Understanding multiprocessing helps you write Python code that scales with modern hardware.

Key Concepts: Multiprocessing in Python

  • Process vs. Thread: Processes have separate memory space; threads share memory. Multiprocessing avoids the GIL.
  • Global Interpreter Lock (GIL): Prevents true parallelism in threads for CPU-bound tasks.
  • Process Pool: Use multiprocessing.Pool to manage a pool of worker processes.
  • Inter-Process Communication (IPC): Use Queues, Pipes, or shared memory to exchange data between processes.
  • Synchronization: Use Locks, Events, and Semaphores to coordinate processes.

Example: Basic Multiprocessing

basic_mp_example.py 
import multiprocessing
import os

def worker(num):
    print(f"Worker {num} in process {os.getpid()}")

if __name__ == '__main__':
    processes = []
    for i in range(4):
        p = multiprocessing.Process(target=worker, args=(i,))
        processes.append(p)
        p.start()
    for p in processes:
        p.join()

Example: Multiprocessing Pool

pool_example.py 
import multiprocessing

def square(n):
    return n * n

if __name__ == '__main__':
    with multiprocessing.Pool(4) as pool:
        results = pool.map(square, range(8))
        print(results)

[0, 1, 4, 9, 16, 25, 36, 49]
