38. Virtual Environments (venv)

Intermediate Level

üì¶Why Virtual Environments?

As you start working on more complex Python projects, you'll inevitably need to install external libraries or packages (e.g., requests, numpy, django). These libraries have specific versions, and different projects might require different versions of the same library. This can lead to conflicts: if Project A needs library_x version 1.0, and Project B needs library_x version 2.0, installing both globally on your system can break one or both projects.

This is where Virtual Environments come in! A virtual environment is an isolated Python environment that allows you to install packages for a specific project without affecting other projects or your system's global Python installation.

üõ†Ô∏èThe venv Module

Python 3 comes with a built-in module called venv (virtual environment) that allows you to create lightweight virtual environments. It's the recommended way to manage project dependencies.

Here's a typical workflow for using venv from your terminal (Command Prompt/PowerShell on Windows, Terminal on macOS/Linux):

  • Create a virtual environment: 
    python3 -m venv myenv
# or 'python' instead of 'python3' depending on your system setup
    This command creates a directory named myenv (you can choose any name) in your current project directory. Inside myenv, you'll find a copy of the Python interpreter and directories for installing packages.
  • Activate the virtual environment:
    • On macOS/Linux: 
      source myenv/bin/activate
    • On Windows (Command Prompt): 
      myenv\Scripts\activate.bat
    • On Windows (PowerShell): 
      myenv\Scripts\Activate.ps1
    Once activated, your terminal prompt will usually change to indicate that you are inside the virtual environment (e.g., (myenv) your_username@your_computer:~/my_project$).
  • Install packages: 
    pip install requests numpy
    Any packages installed now will only reside within your myenv virtual environment.
  • Deactivate the virtual environment: 
    deactivate
    This returns you to your system's global Python environment.

Example: Setting up a Project with venv

terminal_commands.sh (simulated) 
# Navigate to your project directory (if not already there)
cd my_python_project

# Create a virtual environment named 'env'
python3 -m venv env

# Activate the virtual environment
source env/bin/activate  # For macOS/Linux
# env\Scripts\activate.bat  # For Windows Command Prompt
# env\Scripts\Activate.ps1  # For Windows PowerShell

# (env) Now you are inside the virtual environment

# Install a package (e.g., requests)
pip install requests

# Verify installed packages in this environment
pip freeze

# Create a Python script that uses the installed package
# echo 'import requests; response = requests.get("https://www.google.com"); print(response.status_code)' > app.py

# Run the script using the virtual environment's Python
python app.py

# Deactivate the virtual environment when done
deactivate

# (Outside env) Now if you run 'pip freeze' again, 'requests' won't be listed unless it's globally installed.

Note: The code editor above simulates terminal commands. You cannot actually run these commands within this browser-based Python environment. This section is purely for conceptual understanding of how venv works in a real development setup.
