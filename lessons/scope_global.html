21. Scope and Global Variables

Beginner

Understanding Variable Visibility

Scope refers to the region of a program where a variable is accessible. In Python, variables have different scopes:

  • Local Scope: Variables defined inside a function are local to that function. They can only be accessed from within that function.
  • Global Scope: Variables defined outside of any function are global. They can be accessed from anywhere in the program, both inside and outside functions.

Understanding scope is crucial to avoid unexpected behavior and bugs in your programs.

Local vs. Global Example

Local vs. Global Example 
global_message = "I am a global variable!" # Global variable

def my_function():
    local_variable = "I am a local variable." # Local variable
    print(local_variable)
    print(global_message) # Global variable is accessible inside function

my_function()
# print(local_variable) # This would cause an error: NameError, local_variable is not defined

print(global_message) # Global variable is accessible outside function

I am a local variable.

The `global` Keyword

If you want to modify a global variable from inside a function, you must explicitly declare it using the global keyword. If you don't, Python will assume you are creating a new local variable with the same name.

global Keyword Example 
counter = 0 # Global variable

def increment_counter():
    global counter # Declare that we intend to modify the global 'counter'
    counter += 1
    print(f"Inside function: counter is {counter}")

print(f"Before function call: counter is {counter}")
increment_counter()
increment_counter()
print(f"After function calls: counter is {counter}")

print("\n--- Without 'global' keyword (creates a new local variable) ---")
x = 10 # Global x
def modify_x():
    x = 5 # This creates a NEW LOCAL variable named x, doesn't affect global x
    print(f"Inside modify_x: local x is {x}")

print(f"Before modify_x: global x is {x}")
modify_x()
print(f"After modify_x: global x is still {x}")

üì§ Expected Output: 

Before function call: counter is 0
Inside function: counter is 1
Inside function: counter is 2
After function calls: counter is 2

--- Without 'global' keyword (creates a new local variable) ---
Before modify_x: global x is 10
Inside modify_x: local x is 5
After modify_x: global x is still 10
