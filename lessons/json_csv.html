Why JSON and CSV Matter

JSON and CSV are the most common formats for exchanging data between systems, APIs, and applications. Mastering them is essential for data science, web development, and automation.

Real-world scenarios:

  • Consuming and producing API data (JSON)
  • Importing/exporting data to spreadsheets (CSV)
  • Data cleaning and transformation pipelines

Python's json and csv modules make it easy to work with these formats efficiently and safely.

üí°

Key Concepts: JSON & CSV in Python

  • Parsing: Converting JSON/CSV text into Python objects (dicts, lists, etc.).
  • Serialization: Converting Python objects to JSON/CSV text for storage or transmission.
  • Reading/Writing: Using open(), json.load(), json.dump(), csv.reader, and csv.writer.
  • Handling Edge Cases: Dealing with missing values, encoding, and nested data.
  • Third-party Libraries: pandas for advanced data manipulation.

Example: Reading and Writing JSON

json_example.py 
import json

data = {"name": "Alice", "age": 30}

# Serialize to JSON string
json_str = json.dumps(data)
print(json_str)

# Parse JSON string back to dict
parsed = json.loads(json_str)
print(parsed)

Example: Reading and Writing CSV

csv_example.py 
import csv

rows = [
    ["name", "age"],
    ["Alice", 30],
    ["Bob", 25]
]

# Write to CSV string (using StringIO for demo)
from io import StringIO
output = StringIO()
writer = csv.writer(output)
for row in rows:
    writer.writerow(row)
csv_str = output.getvalue()
print(csv_str)

# Read from CSV string
input_io = StringIO(csv_str)
reader = csv.reader(input_io)
for row in reader:
    print(row)

üí° Expected Output:

name,age
Alice,30
Bob,25
['name', 'age']
['Alice', '30']
['Bob', '25']

Handling Nested JSON

Real-world JSON often contains nested objects or lists. To convert these to CSV, you need to flatten the structure.

flatten_json.py 
def flatten(d, parent_key='', sep='_'):
    items = []
    for k, v in d.items():
        new_key = f"{parent_key}{sep}{k}" if parent_key else k
        if isinstance(v, dict):
            items.extend(flatten(v, new_key, sep).items())
        else:
            items.append((new_key, v))
    return dict(items)

nested = {"name": "Alice", "address": {"city": "NY", "zip": "10001"}}
print(flatten(nested))

üí° Output:

{'name': 'Alice', 'address_city': 'NY', 'address_zip': '10001'}

Using Pandas for JSON & CSV

pandas makes it easy to load, manipulate, and convert between JSON and CSV formats, especially for large datasets.

pandas_json_csv.py 
import pandas as pd

df = pd.read_json('[{"name": "Alice", "age": 30}, {"name": "Bob", "age": 25}]')
print(df)
df.to_csv('output.csv', index=False)

Encoding & Unicode Issues

Data files may contain special characters or different encodings. Always specify encoding='utf-8' when reading/writing files, and use ensure_ascii=False for JSON with non-ASCII characters.

encoding_example.py 
import json

data = {"name": "Jos√©"}
json_str = json.dumps(data, ensure_ascii=False)
print(json_str)

üí° Output:

‚úÖ Best Practices for JSON & CSV

  • Always validate and sanitize input data before processing.
  • Use with open(...) to ensure files are closed properly.
  • Specify encoding='utf-8' for file operations.
  • Handle missing values and edge cases explicitly.
  • Use pandas for large or complex data transformations.
  • Document your data schema and assumptions.

‚ö†Ô∏è Common Pitfalls

  • Assuming all JSON is flat‚Äîreal data is often nested.
  • Forgetting to handle encoding‚Äîcan cause errors with non-ASCII data.
  • Not handling missing or extra fields in CSV/JSON.
  • Mixing up data types (e.g., numbers as strings).
  • Not closing files‚Äîuse with statements.
