üñ•Ô∏èPygame: Basic Window & Event Loop

Pygame makes it easy to create windows, draw graphics, and handle events. Here‚Äôs how to open a window and respond to the quit event:

pygame_window_example.py 
import pygame

pygame.init()
screen = pygame.display.set_mode((640, 480))
pygame.display.set_caption("My Game")

running = True
while running:
    for event in pygame.event.get():
        if event.type == pygame.QUIT:
            running = False

pygame.quit()

üí° Expected Output:

A window titled "My Game" opens and closes when you click the close button.

üöÄSprites and Movement

Sprites are objects that can move and interact in your game. Pygame provides a Sprite class to help manage them. Here‚Äôs how to move a rectangle with arrow keys:

sprite_movement_example.py 
import pygame

pygame.init()
screen = pygame.display.set_mode((640, 480))
x, y = 320, 240

running = True
while running:
    for event in pygame.event.get():
        if event.type == pygame.QUIT:
            running = False
    keys = pygame.key.get_pressed()
    if keys[pygame.K_LEFT]: x -= 5
    if keys[pygame.K_RIGHT]: x += 5
    if keys[pygame.K_UP]: y -= 5
    if keys[pygame.K_DOWN]: y += 5
    screen.fill((255, 255, 255))
    pygame.draw.rect(screen, (0, 128, 255), (x, y, 60, 60))
    pygame.display.flip()

pygame.quit()

üí° Expected Output:

A blue rectangle moves with the arrow keys.

üèÜGame State and Scoring

Game state and scoring are key to gameplay. Here‚Äôs how to display a score on the screen:

score_example.py 
import pygame

pygame.init()
screen = pygame.display.set_mode((640, 480))
font = pygame.font.SysFont(None, 48)
score = 0

running = True
while running:
    for event in pygame.event.get():
        if event.type == pygame.QUIT:
            running = False
    screen.fill((255, 255, 255))
    score_text = font.render(f"Score: {score}", True, (0, 0, 0))
    screen.blit(score_text, (250, 220))
    pygame.display.flip()

pygame.quit()

üí° Expected Output:

Score: 0 is displayed in the center of the window.

üñºÔ∏èAdding Images and Assets

Pygame can load and display images (sprites). Here‚Äôs how to load and draw an image:

image_asset_example.py 
import pygame

pygame.init()
screen = pygame.display.set_mode((640, 480))

running = True
while running:
    for event in pygame.event.get():
        if event.type == pygame.QUIT:
            running = False
    screen.fill((255, 255, 255))
    screen.blit(image, (300, 220))
    pygame.display.flip()

pygame.quit()

üí° Expected Output:

The image 'player.png' is displayed at the center of the window.

‚úÖ Best Practices for Game Development

  • Keep your game loop efficient and avoid blocking operations.
  • Organize code using classes for sprites and game objects.
  • Use asset loading (images, sounds) outside the main loop.
  • Test on different screen sizes and platforms.
  • Document controls and gameplay for users.

‚ö†Ô∏è Common Pitfalls

  • Not calling pygame.quit(): Always clean up resources on exit.
  • Ignoring frame rate: Use pygame.time.Clock() to control FPS.
  • Hardcoding values: Use variables for screen size, colors, etc.
  • Not handling user input: Always check for events in the game loop.
