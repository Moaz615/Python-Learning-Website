Why Design Patterns Matter

Design patterns are proven solutions to common software design problems. They help you write code that is more maintainable, flexible, and scalable. Understanding design patterns is essential for building robust, professional-grade Python applications.

Patterns are not code you copy-paste, but reusable templates for solving recurring problems. They make your code easier to understand and modify, and help teams communicate ideas more effectively.

Real-world scenarios:

  • Ensuring only one instance of a class exists (Singleton)
  • Decoupling code for easier testing and extension (Strategy, Observer)
  • Managing object creation (Factory, Builder)
  • Structuring complex systems (Facade, Adapter, Decorator)

Python supports all classic design patterns, often with its own idiomatic twists.

Key Design Patterns in Python

  • Singleton: Ensures a class has only one instance and provides a global point of access to it.
  • Factory: Creates objects without specifying the exact class of object that will be created.
  • Strategy: Lets you define a family of algorithms, put each in a class, and make them interchangeable.
  • Observer: Lets objects notify other objects about changes in their state.
  • Decorator: Adds new functionality to an object dynamically.

Example: Singleton Pattern

singleton.py 
class Singleton:
    _instance = None

    def __new__(cls, *args, **kwargs):
        if cls._instance is None:
            cls._instance = super().__new__(cls)
        return cls._instance

obj1 = Singleton()
obj2 = Singleton()
print(obj1 is obj2)  # True

Example: Strategy Pattern

strategy.py 
class SortStrategy:
    def sort(self, data):
        raise NotImplementedError()

class BubbleSort(SortStrategy):
    def sort(self, data):
        arr = data[:]
        n = len(arr)
        for i in range(n):
            for j in range(0, n - i - 1):
                if arr[j] > arr[j+1]:
                    arr[j], arr[j+1] = arr[j+1], arr[j]
        return arr

class QuickSort(SortStrategy):
    def sort(self, data):
        if len(data) <= 1:
            return data
        pivot = data[0]
        less = [x for x in data[1:] if x <= pivot]
        greater = [x for x in data[1:] if x > pivot]
        return QuickSort().sort(less) + [pivot] + QuickSort().sort(greater)

# Context
class Sorter:
    def __init__(self, strategy):
        self.strategy = strategy
    def sort(self, data):
        return self.strategy.sort(data)

sorter = Sorter(BubbleSort())
print(sorter.sort([5, 2, 9, 1]))  # [1, 2, 5, 9]
sorter = Sorter(QuickSort())
print(sorter.sort([5, 2, 9, 1]))  # [1, 2, 5, 9]
