What is Recursion?

Recursion is a programming technique where a function calls itself, either directly or indirectly, to solve a problem. It's a powerful approach for problems that can be broken down into smaller, self-similar sub-problems.

Every recursive function must have two main parts:

  • Base Case: This is the condition that stops the recursion. Without a base case, the function would call itself indefinitely, leading to an infinite loop and eventually a RecursionError (stack overflow).
  • Recursive Step: This is where the function calls itself with a modified input, moving closer to the base case.
üí°

Why Use Recursion?

Recursion can make code more elegant and easier to understand for certain types of problems that have a naturally recursive structure. Common applications include:

  • Traversing tree-like data structures (e.g., file systems, XML/HTML parsing).
  • Solving mathematical problems like factorial calculation, Fibonacci sequence.
  • Backtracking algorithms (e.g., solving mazes, N-Queens problem).

However, recursion can sometimes be less efficient than iterative solutions (using loops) due to the overhead of function calls and memory usage on the call stack.

Example: Calculating Factorial Recursively

factorial_recursion.py 
# Factorial of a non-negative integer n is the product of all positive integers less than or equal to n.
# Example: 5! = 5 * 4 * 3 * 2 * 1 = 120
# Recursive definition: n! = n * (n-1)!
# Base case: 0! = 1

def factorial(n):
    print(f"Calculating factorial({n})...")
    if n == 0:
        print(f"Base case reached: factorial(0) = 1")
        return 1  # Base case
    else:
        result = n * factorial(n - 1) # Recursive step
        print(f"Returning {n} * factorial({n-1}) = {result}")
        return result

print("\n--- Factorial of 3 ---")
fact_3 = factorial(3)
print(f"Factorial of 3 is: {fact_3}")

print("\n--- Factorial of 5 ---")
fact_5 = factorial(5)
print(f"Factorial of 5 is: {fact_5}")
